# Paper Table Value Provenance

Current as of 2026-07-04. Paths are relative to the repo root. The
`benchmarking/results/…` folders cited below are large and are **not** committed;
restore them with `data_sync/data_sync.sh pull <tag>` (see `DATA.md`). The
`my_paper/paper_figures/…` CSVs are committed.

**Reproduction (2026-07-04):** `python benchmarking/run.py --tier paper` now
runs *every* paper sweep and then invokes `publish_paper_figures.py` +
`make_paper_figures.py`, so all committed `my_paper/paper_figures/*.csv` sources
below are regenerated from that one command. Three tables still read their
numbers from the sweep's raw `benchmarking/results/<dir>/` output (not a
committed CSV) -- flagged **[raw-dir]** below; the run reproduces the dir, but
publishing a committed CSV for them is a TODO.

**Figure coverage:** all 27 `\includegraphics` figures except three are produced
by that one command (25 sweeps + the publish/figure steps; the estimator table
and Figs 8/9 share the manifold eval's `closed_loop_estimator_lhs_runs.csv`).
The `integrated_hero`, `latency_awareness`, and `rollout_diag` sweeps (the last
run with `LOG_MPC_PREDICTIONS=1` so `rollout_prediction_validation.py` has its
`mpc_predictions.npz`) are wired into `run.py`, with copy steps in
`make_paper_figures.py`. **Three figures are NOT in the automated tier:**
- `hil_hud_pov.png` -- driver-POV screenshot; needs the G29 rig + a human driver.
- `teleop_counterfactual_clearance.png` -- aggregate of real human drives via
  `aggregate_teleop_counterfactual.py --dirs <human convoy_counterfactual runs>`.
- `speed_profile_gg.png` -- its `make_fig_speed_profile.py` reads a bespoke
  *paired* per-(path,terrain,speed) CSV (`cte_baseline`/`cte_terrain` columns
  over multiple paths), not the long `summary_by_variant.csv` that
  `speed_profile_ablation.py` currently emits. Regenerate manually by running
  `speed_profile_ablation.py` over the paper's path set and pivoting its results
  into `my_paper/paper_figures/speed_profile_ablation.csv`. Wiring this cleanly
  into `run.py` is a TODO (needs the script to emit the paired schema directly).

This sheet is a source map for the numerical tables in
`my_paper/paper.tex`. For a fresh submission pass, regenerate the publish
artifacts first:

```bash
python benchmarking/publish_paper_figures.py
python benchmarking/plot_compact_heatmaps.py
```

If the underlying Chrono data are stale, run the relevant sweep before
publishing. The main autonomous paper tier is driven by:

```bash
python benchmarking/run.py --tier paper
```

## Known Caveats

- Some result CSVs include a small number of non-`ok` or blank-KPI rows. The
  paper captions now describe scheduled matrix size where appropriate and use
  valid KPI rows for medians.
- Human-in-the-loop rounds are intentionally outside the automated paper tier;
  HIL figures/protocol demonstrate capability, not a human-subjects safety
  comparison.
- Archived MPPI/SLSQP-NMPC rows exist in old `benchmarking/results/*` folders.
  Current published safety and latency artifacts filter to no-filter and
  DOB-CBF variants only.
